hexo博客搭建之路与在配置Bufferfly上踩的坑（2)

发表于2022-07-08|更新于2022-07-20|博客搭建与主题配置
|阅读量:

原本在搭建的过程顺风顺水，以为很快就可以搭好，结果在配置主题的路上越走越远，这里要说明的是除了butterfly的文档外我大部分网上找的都是对应3.8左右的版本，但是在近年 5 月份的时候他已经更新到了4.0的版本了所以有一些东西的配置跟之前就有所出入了,我所配的版本已经是4.3.0了。我写的逻辑是按页面来可划分的，暂时就是主页与文章页两个模块

Butterfly配置

对于主题的配置都是在主题文件下的_config.yml文件中进行的

首页配置

顶部图

如果不要顯示頂部圖，可直接配置 disable_top_img: true

属性名（解释） 属性值
index_img（首页的顶部图） 图片地址

网站副标题

可设置主页中显示的网站副标题或者喜欢的座右铭。

修改source即可

必须是 /xxx/，后面||分开，然后写图标名。

如果不希望显示图标，图标名可不写。

默认子目录是展开的，如果你想要隐藏，在子目录里添加 hide 。

文章置顶

【推荐】hexo-generator-index从 2.0.0 开始，已经支持文章置顶功能。你可以直接在文章的front-matter区域里添加sticky: 1属性来把这篇文章置顶。数值越大，置顶的优先级越大。

修改 主题配置文件

图片懒加载

4.0版本之后图片懒加载的属性有一些变化，作者把懒加载的图片默认设置为透明，如果要自定义懒加载图片就在placeholder属性填写懒加载的图片路径

页面加载动画

当进入网页时，因为加载速度的问题，可能会导致top_img图片出现断层显示，或者网页加载不全而出现等待时间，开启preloader后，会显示加载动画，等页面加载完，加载动画会消失。

文章页

Front-matter

Front-matter 是 markdown 文件最上方以 — 分隔的区域，用于指定个别档案的变数。

写法 解释
title 【必需】文章标题
date 【必需】文章创建日期
updated 【可选】文章更新日期
tags 【可选】文章标籤，多标签采用数组形式，如tags: [css,js]
categories 【可选】文章分类
keywords 【可选】文章关键字
top_img 【可选】文章顶部图片
cover 【可选】文章缩略图(如果没有设置top_img,文章页顶部将显示缩略图，可设为false/图片地址/留空)
comments 【可选】显示文章评论模块(默认 true)

文章置顶

【推荐】hexo-generator-index从 2.0.0 开始，已经支持文章置顶功能。你可以直接在文章的front-matter区域里添加sticky: 1属性来把这篇文章置顶。数值越大，置顶的优先级越大。
